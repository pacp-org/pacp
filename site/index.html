Visão geral

O que o PACP resolve

Elimina planilhas ad-hoc e integrações proprietárias entre ERPs, e-commerces, configuradores e orçamentistas.

📋

Catálogo como contrato

Dois tipos de documento JSON — CATALOG (manifesto com regras) e PRODUCT (por arquivo). Estrutura clara, validável por schema.

🧮

Motor de regras, não variantes

Modele atributos, opções e regras condicionais. O preço é calculado sob demanda — sem explodir milhares de SKUs.

🎯

Determinístico

Mesma entrada, mesmo resultado. Ordem de regras, desempate e fallbacks são definidos de forma normativa.

📦

Produto autocontido

SKU, fabricante, marca, imagens, peso, dimensões — tudo no JSON do produto. Sem depender de PIM externo.

🔧

Extensível por vertical

Extension profiles padronizam campos específicos de cada setor (móveis, pisos, iluminação) com validação formal.

✅

Validação automática

CLI que valida schema, referências, semântica de operações, lote, unidade e campos de profiles.

Arquitetura

Estrutura de um catálogo PACP

Um catálogo é um manifesto JSON que referencia produtos, regras de precificação, tabelas e profiles.

Manifesto CATALOG

{
  "spec": "1.0.0",
  "document_type": "CATALOG",
  "catalog": { "id": "cat_moveis_2026", "name": "Catálogo Móveis 2026" },
  "profiles": ["moveis", "fiscal-br"],
  "product_refs": [
    { "id": "prod_painel", "path": "products/prod_painel.json" }
  ],
  "rulesets": [
    {
      "id": "rs_base", "target": "BASE",
      "rules": [
        { "id": "rule_material", "operation": "LOOKUP", "tableId": "tbl_material" }
      ]
    }
  ],
  "tables": [ ... ],
  "constraints": [ ... ]
}

Documento PRODUCT

{
  "spec": "1.0.0",
  "document_type": "PRODUCT",
  "catalog_id": "cat_moveis_2026",
  "product": {
    "id": "prod_painel",
    "name": "Painel Ripado Elegance",
    "sku": "PNL-ELG-001",
    "manufacturer": "Moveis Artisan",
    "brand": "Artisan Home",
    "description": "Painel ripado em MDF com acabamento nogueira.",
    "gtin": "7891234567890",
    "base_price": 100,
    "images": [{ "url": "https://cdn.ex/pnl-001.jpg", "type": "MAIN" }],
    "weight": { "value": 12.5, "unit": "kg" },
    "dimensions": { "width": 180, "height": 90, "depth": 3, "unit": "cm" },
    "options": [
      { "id": "opt_pvc", "attributeId": "material", "value": "PVC" }
    ],
    "x-finish": "Nogueira natural",
    "x-warranty_months": 12
  }
}

Precificação

Três estágios, dez operações

Regras atuam em ordem determinística sobre BASE, SUBTOTAL e TOTAL.

BASE

Ajustes no preço base: ADD, OVERRIDE, LOOKUP por material, tamanho, etc.

SUBTOTAL

Agregadores e componentes: MAX_OF, MIN_OF, PICK entre alternativas.

TOTAL

Margem, impostos, arredondamento: PERCENT_OF, ROUND, CAP, FLOOR.

Operações do motor

ADD          soma valor fixo             value: 10
PERCENT_OF   percentual sobre alvo       percent: 20
OVERRIDE     substitui valor corrente    value: 50
LOOKUP       busca em tabela matricial   tableId: "tbl_preco"
MAX_OF       maior entre componentes     components: [...]
MIN_OF       menor entre componentes     components: [...]
PICK         primeiro elegível           components: [...]
ROUND        arredondamento              precision: 2
CAP          teto máximo                 max: 500
FLOOR        piso mínimo                 min: 10

Primeiros passos

Como integrar

Do catálogo mínimo ao completo em 6 passos.

Catálogo mínimo

1 produto, 1 regra ADD. Valide com o CLI.

Campos descritivos

Adicione sku, manufacturer, brand, images, weight, dimensions aos produtos.

Declare profiles

Adicione "profiles": ["moveis"]" e use os campos x-* do setor.

Tabelas e condicionais

Use LOOKUP para preços matriciais e when para regras condicionais.

Constraints e dependencies

Modele bloqueios de combinação e relações entre opções.

Múltiplas listas de preço

Ative price_lists para varejo, B2B, atacado via context.

Validador CLI

cd tools/validator
npm ci && npm run build

# validar um arquivo
npm run validate -- ../../spec/1.0.0/examples/moveis/max_of.json

# validar todos os exemplos oficiais
npm run validate:examples
