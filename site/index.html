O problema

Você já viveu isso?

Toda vez que um fabricante manda uma tabela de preços, começa a dor de cabeça. Formatos diferentes, regras escondidas em planilhas, cálculos que ninguém consegue reproduzir. O PACP nasceu para acabar com isso.

Sem o PACP

  • Cada fornecedor manda a tabela em um formato diferente
  • Regras de preço escondidas em células de Excel
  • Dois sistemas calculam preços diferentes para o mesmo produto
  • Milhares de SKUs para cobrir todas as combinações
  • Integração ponto a ponto entre cada par de sistemas

Com o PACP

  • Um formato JSON padronizado para todos
  • Regras de preço explícitas e auditáveis
  • Resultado idêntico em qualquer sistema — determinístico
  • Motor de regras calcula sob demanda — sem explodir SKUs
  • Qualquer sistema lê o mesmo contrato

Como funciona

Um contrato inteligente para seu catálogo

Em vez de planilhas soltas, você descreve seu catálogo e suas regras de preço em arquivos JSON validáveis. Simples para quem lê, poderoso para quem integra.

Catálogo como contrato

Um arquivo JSON descreve o catálogo inteiro: produtos, regras de preço, tabelas e restrições. Qualquer sistema consegue ler.

CATALOG + PRODUCT docs

Preço calculado, não cadastrado

Em vez de criar milhares de variações no cadastro, você define regras que calculam o preço na hora, conforme as opções do cliente.

Rulesets + operations

Sempre o mesmo resultado

A spec define a ordem exata de aplicação das regras, desempates e fallbacks. Dois sistemas diferentes vão chegar ao mesmo preço.

Determinístico por spec

Produto completo no JSON

Nome, SKU, fabricante, marca, fotos, peso, dimensões — tudo dentro do arquivo do produto. Sem precisar de sistema externo.

sku, gtin, images, weight...

Extensível por setor

Campos específicos de cada indústria — móveis, iluminação, pisos, fiscal — são padronizados via Extension Profiles.

profiles: ["moveis"]

Validação automática

Um CLI valida tudo: schema, referências entre arquivos, lote, unidade de venda e campos de profiles. Erros aparecem antes, não depois.

pacp-validate

Arquitetura

Dois tipos de documento, um catálogo completo

O catálogo é um manifesto JSON que referencia os produtos. Cada produto vive no seu próprio arquivo — fácil de versionar, fácil de validar.

catalogo.json 
{
  "spec": "1.0.0",
  "document_type": "CATALOG",
  "catalog": {
    "id": "cat_moveis_2026",
    "name": "Catálogo Móveis 2026"
  },
  "profiles": ["moveis", "fiscal-br"],
  "product_refs": [
    { "id": "prod_painel",
      "path": "products/prod_painel.json" }
  ],
  "rulesets": [ ... ],
  "tables": [ ... ],
  "constraints": [ ... ]
}
products/prod_painel.json 
{
  "spec": "1.0.0",
  "document_type": "PRODUCT",
  "catalog_id": "cat_moveis_2026",
  "product": {
    "id": "prod_painel",
    "name": "Painel Ripado Elegance",
    "sku": "PNL-ELG-001",
    "base_price": 100,
    "weight": { "value": 12.5, "unit": "kg" },
    "x-finish": "Nogueira natural",
    "x-warranty_months": 12
  }
}

Motor de preços

Três estágios, dez operações

O preço passa por um pipeline com ordem definida. Cada estágio aplica um tipo de ajuste. Não tem mágica: a spec define tudo.

Estágio 1

BASE

Ajustes diretos no preço base do produto: somar valor, substituir, buscar em tabela por material ou tamanho.

Estágio 2

SUBTOTAL

Agregadores que comparam componentes: pegar o maior preço, o menor, ou o primeiro que se encaixe.

Estágio 3

TOTAL

Ajustes finais: margem percentual, impostos, arredondamento, teto máximo e piso mínimo.

Primeiros passos

Do zero ao catálogo completo

Quer integrar do jeito mais rápido?

A Hoop é a maior empresa de tecnologia do setor na América Latina. Eles têm uma engine intuitiva de tradução dos seus dados atuais — planilhas, banco de dados, PDFs — para PACP. A plataforma já conecta indústria, representantes e lojas com catálogos atualizados em tempo real.

Conhecer a Hoop

Integrar com o PACP é progressivo. Comece com o mínimo e vá adicionando conforme precisar. Em 6 etapas você tem um catálogo completo.

Comece simples

Crie um catálogo com 1 produto e 1 regra de preço. Valide com o CLI para garantir que está tudo certo.

Descreva seus produtos

Adicione nome, SKU, fabricante, marca, fotos, peso e dimensões. O produto fica completo e autocontido.

Ative os Profiles do seu setor

Declare "profiles": ["moveis"] e use os campos específicos da sua indústria com validação automática.

Tabelas e condicionais

Use LOOKUP para preços que variam por combinação (material x tamanho) e regras condicionais com when.

Restrições e dependências

Bloqueie combinações inválidas com constraints e crie relações entre opções com dependencies.

Múltiplas listas de preço

Ative listas para varejo, B2B e atacado. O contexto define qual usar na hora do cálculo.
